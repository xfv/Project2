問題描述

        一般的相機沒辦法捕捉人眼的廣角畫面，因此景物的重現仰賴多張照片的疊合，重現身歷其境的視覺效果。
在本次作業中，我們捕捉了360度的環景圖(panorama)，且為符合人的視覺喜好，補償 Drift 現象為方形。

學習目標

  • 使用 Harris Corner Algorithm 對影像找出特徵點(feature point)
  • 分別對特徵點取 descriptor 做配對(matching)
  • 使用 RANSAC 演算法找出配對的特徵點(matched pairs)中的inliers解轉換矩陣(transformation matrix)
  • 在影像重疊之處使用下列兩種 blending 方法將影像組合起來
         - Linear Blending
         - Poisson Image Editing
  • 補償影像 drift 現象將影像轉為方形

演算法

  • Feature Detection
          - 先對影像做輕微的 Gaussian Blur
          - 使用 Harris Corner 演算法計算，先算出x和y方向上的微分(derivatives)得到Ix, Iy
          - 計算Ixx = Ix * Ix, Iyy = Iy * Iy, Ixy = Ix * Iy
          - 對Ixx, Iyy, Ixy做 Gaussian Blur得到各個pixel的和Sxx, Syy, Sxy
          - 定義M = [[Sxx, Sxy],
                             [Sxy, Syy]]
          - R = det(M) - k(trace(M))^2
          - R越大的地方越可能是Feature Point
          - 定義Threshold，取R>Threshold的點為Feature Point
  • Descriptor
        - 經過測試發現取window size = 40*40，每5個點取樣一次得到 8*8 的descriptor效果比較好
  • Matching
        - 計算鄰近兩張圖的feature matching
        - 分別對兩張圖的每一個feature point找其最相似的對應點，若其互相匹配則形成一組pair
  • RANSAC
       - 隨機取4個組pair解轉換矩陣M
        - 對其餘每個點皆乘上此轉換矩陣，定義threshold，統計轉換後與對應點的距離小於此值的point（即inliers）個數
        - 遞迴多次後，取最多inliers的那組解
  • Blending
         Method 1: Linear Blending
         - 對x方向產生線性的遮罩(mask)
         - 在組合圖時將重疊部分乘上這個遮罩，即可完成Blending

         Method 2: Poisson Blending
         - 對於所有重疊部份的點fp，和其上下左右四個鄰居fq，計算Vpq = fp-fq
         - 建立A = poisson sparse matrix
         - B = 每個fp對應的Vpq之和形成的陣列
         - 解出Ax = B ，x即為各個fp新的值
         - 將新的fp貼在對應的位置即可完成Blending

  • Drift
         - 先將第一張圖貼在最後一張圖的後面(全景圖必須是360度)
         - 基於第一張圖在頭尾的相對高度應該要一樣，因此計算兩張圖的高度差dy
         - 將dy分散至x軸，得到a = dy/x_max
         - 建立Homography，y' = ax + y


實作細節

  • 程式語言: python
  • 利用的 module:
          - opencv for python
          - math, bumpy, random
          - scipy
          - pyamg
  • Python 較 C++ 來說較為高階，因此實作上較方便，且 opencv for python 的底層一樣是用 C++ 來實做，再加上 numpy 的使用，讓我們可以對矩陣做快速的運算。因此用Python可以說是融合C++的速度，加上python的實作方便性，還有Matlab一樣的矩陣運算，一舉數得。
  • 在 Matching 步驟中，我們是利用 opencv 的 SVD 來解 Transformation Matrix，而在 Poisson Blending 步驟中，因為 Overdetermined system 超出opencv接受的大小，改用 pyamg 來解重疊部分的 pixel value
